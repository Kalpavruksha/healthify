# Security Policy 🔒
 
## Supported Versions
 
| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes    |
| < 1.0   | ❌ No     |

## 🛡️ Reporting a Vulnerability

If you discover a security vulnerability in **Healthify**, please report it responsibly:

1. **Do NOT** open a public GitHub Issue for security flaws
2. Email: **beshu4959gowdaman@gmail.com**
3. Include description, reproduction steps, and potential impact

We will respond within **48 hours** and patch critical issues within **7 days**.

## 🔐 Security Best Practices

- Never commit `.env` files or API keys
- Use environment variables for all credentials
- Validate all user health data inputs server-side
- Keep dependencies updated (`npm audit`)
